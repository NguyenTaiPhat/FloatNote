# Contributing to FloatNote

Thank you for your interest in contributing to FloatNote!

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Run development: `npm run dev`

## Development Guidelines

### Code Style

- Use TypeScript strict mode
- Follow ESLint rules
- Use Prettier for formatting
- Write meaningful commit messages

### Commit Messages

Format: `type: description`

Types:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `style:` Formatting
- `refactor:` Code restructuring
- `test:` Tests
- `chore:` Maintenance

Example: `feat: add twitter card support`

### Pull Requests

1. Create feature branch: `git checkout -b feature/amazing-feature`
2. Make changes
3. Test thoroughly
4. Commit: `git commit -m "feat: add amazing feature"`
5. Push: `git push origin feature/amazing-feature`
6. Open Pull Request

### Testing

- Test on Windows, macOS, and Linux
- Verify all card types work
- Check performance
- Test keyboard shortcuts
- Verify database operations

### Adding New Card Type

See DEVELOPMENT.md for detailed guide.

### Code Review

All submissions require review. We use GitHub pull requests for this.
